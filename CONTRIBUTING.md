# Contributing

Thank you for contributing to the Property Management Platform.

## Before Starting

1. Confirm the requirement and acceptance criteria.
2. Check for an existing issue or create one.
3. Discuss changes that affect architecture, financial behavior, permissions or public APIs.
4. Create a branch following `docs/development/git-workflow.md`.

## Development Expectations

- Keep modules isolated and naming consistent with the product vocabulary.
- Enforce organization and building scope in backend code.
- Do not use floating-point values for money.
- Do not hard-delete financial, lease or audit history.
- Add tests for business rules and authorization.
- Keep environment-specific values outside source control.

## Pull Request Checklist

Use the repository pull request template and verify the definition of done before requesting review.

## Reporting Problems

Use the bug report template for normal defects. Follow `SECURITY.md` for security issues.
