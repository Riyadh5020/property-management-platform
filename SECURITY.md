# Security Policy

## Reporting a Vulnerability

Do not disclose suspected vulnerabilities in a public issue. Report them privately to the project owner or designated security contact.

Include:

- Affected feature or endpoint
- Reproduction steps
- Expected and actual behavior
- Potential impact
- Screenshots or logs with credentials and personal data removed

## Sensitive Data

The platform may process identity documents, agreements, visitor information, incident reports and financial records. Contributors must:

- Use synthetic development and test data.
- Avoid placing personal information in issues, logs or screenshots.
- Never commit production exports or uploaded documents.
- Apply least-privilege access.
- Encrypt data in transit and at rest.
- Keep credentials in an approved secrets manager.

## Supported Versions

Supported production versions and security patch policy will be documented before the first public or client release.
